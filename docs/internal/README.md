# docs/internal/ — Notas de evolução (histórico)

Documentos **internos de planejamento e auditoria**, preservados como registro
histórico da evolução do EIGAN. Não são a documentação de referência do produto
— para essa, veja [`../README.md`](../README.md) (arquitetura, ADRs, roadmap).

São *retratos de um momento* (podem conter dados desatualizados em relação ao
estado atual do código) e existem para rastreabilidade das decisões:

- [`AUDIT.md`](AUDIT.md) — auditoria arquitetural do ponto de partida (PASSO 0),
  base para os ADRs em [`../adr/`](../adr/).
- [`CURRENT_STATE.md`](CURRENT_STATE.md) — retrato do estado do projeto num marco
  de reconhecimento.
- [`GAP_ANALYSIS.md`](GAP_ANALYSIS.md) — matriz de capacidades (o que existe × o
  que falta) num marco de reconhecimento.
- [`TARGET_ARCHITECTURE.md`](TARGET_ARCHITECTURE.md) — visão de arquitetura-alvo
  de longo prazo.

> A fonte de verdade viva é o código, os [ADRs](../adr/) e o
> [ROADMAP](../ROADMAP.md). Se um fato aqui divergir dessas fontes, elas vencem.
